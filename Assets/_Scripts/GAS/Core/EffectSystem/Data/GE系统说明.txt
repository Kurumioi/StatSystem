层数存法（配置 StackAggregation）

一、共享层数 SharedCount（默认）
同一 Data 一份 Runtime
StackCount = 层数倍率
RemainingDuration / RemainingPeriod 整份共享
每次 Apply：StackCount += 每次应用增加层数  受最大堆叠层数卡住
首次 Apply 也按该值写入初始层数
叠层不会长出第二条独立时间线

二、各层独立 IndependentInstances
同一 Data 可并存多份 Runtime 每份 StackCount=1 各自寿命与周期
每次 Apply：一次新建「每次应用增加层数」份  受剩余空位卡住
已满 → 拒绝（返回 null）

配置
层数存法
最大堆叠层数
每次应用增加层数（例 =2 → 共享 +2 层 / 独立一次建 2 份）
再应用：刷新时长 / 重置周期 / 到期处理

不可叠 = 最大堆叠层数设为 1
可叠 = 最大堆叠层数 > 1 且每次应用增加层数通常为 1

每帧 UpdateGE（两种存法共用）
1 先扣寿命 到期则 HandleExpiration 本帧不再跳伤
2 未到期再扣周期 到点跳伤并 ResetPeriod

刷新时长 vs 重置周期（主要给共享层数的再应用用）
刷新时长 动 RemainingDuration  Buff还活多久 → 续命
重置周期 动 RemainingPeriod    下次跳伤还有多久 → 重排跳伤拍子
两个开关互不绑死 可只开一个

========== 共享层数下的四种再应用组合 ==========

组合一：可堆叠 + 重置周期
例：叠层蓄力类（文档举例 火男被动叠层）
叠新层时重置周期计时器 跳伤拍子重来

组合二：可堆叠 + 不重置周期
例：撕裂类 Dot（共享时钟 + 伤害×层数）
假设：每2秒造成 层数×10 点伤害 总时长4s 最多叠5层
且 DurationRefresh=不刷新  PeriodReset=不重置

t=0 上第1层  Stack=1  周期倒计时开始
t=1 叠到第2层 Stack=2  周期不重置 接着原来的倒计时跑
t=2 第一次跳伤 = 2×10 = 20  然后周期重新计2秒
t=4 第二次跳伤 = 20  同时整份 GE 到期移除

结论：始终只有一条共享时钟 跳几次由这份周期决定 伤害按当前层数放大
不是每层各自再开一条 t=2/4 和 t=3/5 的时间线

组合三：不可堆叠 + 重置周期
例:中毒效果
中毒总时长持续5秒，每2秒跳一次伤害
那么会在t=2,4时触发
玩家中毒后，如果在t=3时重新施加效果,则刷新整个中毒效果
总时长 t = 3+5 = 8s
那么t = 3+2 =5s , t=5+2=7s 时触发

组合四：不可堆叠 + 不重置周期
例：点燃效果
总时间持续5秒，每2秒跳一次伤害
第一次点燃在t=0，第一次伤害t=2，第二次伤害t=4，结束t=5
如果在t=3再次点燃，持续时间刷新为5秒，新结束时间t=8
但周期不重置，原定t=4的伤害仍然会触发，之后周期继续，下一次伤害在t=6

========== 各层独立举例 ==========

例：每刀一道伤口 period=2 duration=4
t=0 伤口A 周期开始
t=1 伤口B 新建 周期自己从满开始
t=2 A跳伤10  B还在倒计时
t=3 B跳伤10
A到点独自消失 B继续
